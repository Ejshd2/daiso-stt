# 07. 텍스트 정규화(전처리) 최소 룰

## 목적
검색/분류가 잘 되도록 STT 결과를 “검색 친화형”으로 정리한다.
(과도한 LLM 보정은 비용/지연 증가 → PoC에서는 규칙 기반 우선)

## 필수 룰(권장)
1) 필러 제거
- “음…”, “어…”, “그…”, “저기…” 등

2) 공백/반복 정리
- 연속 공백 → 단일 공백
- 반복 문자(“매트트트”) → 축약

3) 간단 치환(도메인 동의어)
- “물 안 미끄럽게” → “미끄럼방지”
- “문틈 바람 막는” → “문풍지” (데이터/카탈로그가 지원할 때만)

4) 정규화 결과가 빈 문자열이 되는 경우, normalized_text는 text_raw를 그대로 사용한다(과도한 정규화 방지).

## 주의
- 정규화가 과하면 원문 의미가 깨질 수 있음
- 변경은 반드시 로그에 normalized_text로 분리 저장
