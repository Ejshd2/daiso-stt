# 03. 공통 지침 (Vibe Coding 운영 규칙)
PoC 1차 단계에서는 동일 조건 비교를 위해 음성 파일 기반 STT만을 사용하며,
실시간 Streaming 연동은 2차 확장 범위로 정의한다.


## 1) 계약 고정
- `04_INTERFACE_CONTRACT.md`의 입출력 스키마는 임의 변경 금지
- 변경 시 `10_DECISION_LOG.md`에 반드시 기록(사유/영향/대안)

## 2) 속도 우선 루프
- “작게 구현 → 배치 테스트 → 수치 확인 → 임계값 조정” 반복
- 기능 추가보다 **재현성/측정 가능성** 우선

## 3) 안전 우선
- 재요청 1회 이후에도 불가한 경우 FAIL 처리
- 직원 개입 없이 자동 종료를 원칙으로 한다

## 4) 로그는 제품이다
- STT 결과 자체보다, “왜 재요청/실패했는지”가 남아야 PoC가 된다.
